🔹 1. vipp[0] colorspace set and get not same [7-33]
log
编辑
W0101 01:06:47.817776  1247 videoInputHw.c:901] <videoInputHw_SetChnAttr> Be careful! vipp[0] colorspace set and get not same[7-33]
含义：
应用程序设置的颜色空间（colorspace）为 7，但驱动实际返回的是 33，两者不一致。
可能原因：
驱动或硬件不支持你请求的颜色格式（如 YUV422 vs YUV420）
配置文件中颜色空间参数写错
影响：
可能导致图像偏色、花屏，或后续编码异常。
✅ 建议：
检查你的视频通道属性配置（如 AW_MPI_VI_SetChnAttr），确保 enPixelFormat 和 enCompressMode 与传感器/ISP 支持的格式匹配。
🔹 2. ISP0 maybe online mode, ignore tdm open failed
log
编辑
W0101 01:06:47.820306  1247 isp_dev.c:454] <isp_dev_open> [ISP_WARN]ISP0 maybe online mode, ignore tdm open failed.
含义：
ISP 工作在 online 模式（直通模式，sensor → ISP → VI，无 DDR 缓存），因此不需要 TDM（Tile Data Manager，用于 offline 模式的数据搬运）。
这是正常现象，不是错误。
✅ 结论：可忽略，说明 ISP 配置为高性能低延迟模式。
🔹 3. open ... isp0_..._ctx_saved.bin failed
log
编辑
W0101 01:06:47.834378  1247 isp_helper.c:146] <isp_ctx_save_init> [ISP_WARN]open /mnt/extsd/isp0_1920_1080_30_0_gc2083_mipi_ctx_saved.bin failed, err:No such file or directory.
含义：
ISP 尝试从 SD 卡加载 之前保存的校准/调优参数（context 文件），但文件不存在。
背景：
全志 ISP 支持将自动曝光、白平衡等状态保存到文件，下次启动时快速恢复。
✅ 结论：首次运行或未保存过上下文时正常，不影响基本功能。
如果你想提升启动速度或保持图像一致性，可以：
运行一次后手动保存 context
或在应用退出时调用 AW_MPI_ISP_SaveContext()
🔹 4. cannot find gc2083_mipi_1920_1080_30_0_0 isp config...
log
编辑
W0101 01:06:47.839520  1247 isp_ini_parse.c:1628] <parser_ini_info> [ISP_WARN]cannot find gc2083_mipi_1920_1080_30_0_0 isp config, use gc2083_mipi_1920_1088_15_0_0 -> [gc2083_mipi_isp603_20241205_171656_final_rgb_suit]
含义：
系统找不到 精确匹配当前分辨率/帧率（1920x1080@30fps）的 ISP 配置文件，于是降级使用了另一个配置（1920x1088@15fps 的配置）。
为什么是 1088？
全志 ISP 要求高度为 16 对齐，1080 → 1088 是常见 padding。
风险：
自动曝光/白平衡可能不准
图像锐化、降噪效果非最优
极端情况下可能出现绿边、过曝等
✅ 建议：
在 ISP 配置目录（通常是 /etc/isp/ 或 /mnt/extsd/isp/）中添加对应分辨率的 .ini 配置文件
文件名应为：gc2083_mipi_1920_1080_30_0_0.ini
可从 SDK 中找模板，或用 ISP tuning 工具生成
